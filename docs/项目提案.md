项目提案：Compute-Sentry —— K8s 原生异构算力确定性治理平台
# 1. 项目概述 (Executive Summary)
Compute-Sentry 是一款面向大规模 AI 训练/推理集群的平台级项目。通过在 Kubernetes 原生调度与容器运行时刻（Runtime）引入非侵入式劫持与确定性预检技术，解决万卡集群在分布式环境下“性能黑盒”与“亚健康节点引发任务崩溃”的行业顽疾。

# 2. 核心痛点 (Problem Statement)

硬件亚健康 (Sub-health)： 显卡虽处于 Ready 状态，但因 PCIE 降级或隐性计算错误（SDC）导致模型 Loss 爆炸或训练大幅减速。

观测盲区 (Observability Gaps)： 传统监控无法捕捉到微秒级的 NCCL 通信抖动，导致长尾延迟（Stragglers）难以定界。

调度失效 (Topology Blindness)： 静态拓扑调度无法应对动态的网络拥塞，导致跨交换机通信成为性能瓶颈。

# 3. 设计原则与价值观 (Design Principles)
Zero-Friction（零摩擦）： 算法研发无需修改任何 Python 代码，运维无需升级 Linux 内核。

Determinism First（确定性优先）： 宁可任务在队列等待，也不允许带有隐患的算力进入生产环境。

Full-Stack Insight（全栈洞察）： 跨越 C++ 用户态、Go 控制面与 TS 视图层，实现硬件状态与业务逻辑的深度映射。

# 4. 架构设计 (Architecture)
## 4.1 技术栈分布
感知层 (C++)： libCompute-Sentry-Spy.so。利用 LD_PRELOAD 劫持 libcuda 和 libnccl。

控制层 (Go)： K8s Operator & Mutating Webhook。负责资源生命周期管理。

策略层 (Python)： 异常检测模型与算力评估算法。

展示层 (TypeScript)： 3D 物理拓扑孪生看板。

# 5. 关键业务逻辑 (Workflow)
## 5.1 确定性准入流 (Deterministic Admission)
动态注入： 当 PyTorchJob 等 CRD 提交时，Webhook 自动在容器环境变量中注入 LD_PRELOAD 路径。

强制预检： Pod 调度后，业务代码启动前，由 Compute-Sentry-Agent 拉起预检算子进行 HBM/P2P 压力测试。

熔断保护： 测试未达标节点自动触发 Taint 标记，并由 Operator 重调度 Pod。

## 5.2 透明可观测流 (Observability Pipeline)
指标劫持： Spy 库实时统计 NCCL 算子耗时，通过 UDS (Unix Domain Socket) 零拷贝传输至 Agent。

拓扑关联： 将采集到的延迟数据与交换机物理端口关联，识别出真实的链路拥塞点。

# 6. 预期收益 (Expected ROI)
MFU 提升： 减少由于慢卡和网络抖动导致的计算等待，预计提升集群利用率 10% 以上。

稳定性增强： 提前拦截 90% 以上的硬件诱发型 NaN 故障。

运维提效： 实现千卡集群的故障分钟级定界，极大降低 Infra 团队排查压力。
